# Setup and Teardown Scripts

OpenCode should provide two bash scripts in a `scripts/` directory to manage the lifecycle of the system and the test VM.

## 1\. `scripts/setup.sh`

This script assumes a fresh Mac and automates the onboarding process:

-   Verifies `/var/lib/sops-nix/key.txt` exists (prompts user to create/place it if missing).
-   Checks for Nix installation.
-   Executes the initial `darwin-rebuild switch --flake .#m1-server`.
-   Prompts the user to ensure Auto-Login is enabled in macOS settings.
-   Prompts the user to manually enable Remote Login (SSH) via System Settings -> General -> Sharing.
-   Prompts the user with a WARNING to turn OFF FileVault, as disk encryption will block headless booting after a power failure.

## 2\. `scripts/teardown.sh`

This script safely spins down the VM and removes it so the user can wipe the state and test again.

-   Unloads the LaunchAgent: `launchctl unload ~/Library/LaunchAgents/org.nixos.lima-test-vm.plist` (or the equivalent label generated by Home Manager).
-   Stops the VM: `limactl stop test-vm`.
-   Deletes the VM disk and state: `limactl delete test-vm`.
-   Cleans up the Tailscale state for the VM from the admin console (optional/printed as a reminder to the user).
